# 배재Pick 2.0 v4.0 Department UI Fix Plan

## Purpose

This document prepares the next UI correction stage after screenshot QA.

v4.0 should not randomly add features.

It should fix the actual gaps found from screenshots.

## Expected Fix Areas

### 1. Department Main Visual

- improve hero card layout
- reduce text density
- improve Nasumi placement
- improve search/filter spacing

### 2. Department Cards

- add clearer category badge
- reduce repeated ProfileRow density
- improve CTA button placement
- make status label less noisy

### 3. Department Detail

- split long sections
- improve section hierarchy
- make career and learning topics easier to scan
- improve Nasumi / QR mission visual card

### 4. Screenshot Readiness

- better first-screen composition
- stronger title and subtitle
- more attractive cards
- less developer-looking text

## Do Not Add Yet

- real interviews
- admin editing
- official department data sync
- authentication-based contribution
- backend storage

## v4.0 Definition of Done

- actual screenshots reviewed
- top P0/P1 visual gaps fixed
- debug APK builds successfully
- screenshots are ready to show to professors/student councils
- README and QA docs updated

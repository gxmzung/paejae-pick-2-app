# 배재Pick 2.0 서버비 및 개인정보 계획

## 1. 서버비 원칙

v1.0에서는 서버비를 최소화한다.

초기 목표:

```text
서버비: 0원 ~ 월 1만 원 이하
v1.0에서는 대부분의 데이터를 앱 내부 mock 또는 로컬 저장으로 처리한다.

2. v1.0 데이터 저장 방식
로컬 저장

SharedPreferences를 사용한다.

저장 대상:

획득 카드
미션 완료 상태
메뉴 카드 획득 상태
관심 동아리 상태
서버 저장 없음

v1.0에서는 서버에 개인별 데이터를 저장하지 않는다.

3. 서버 도입 단계
v1.0
서버 없음 또는 정적 JSON
로컬 저장
개인정보 최소화
v1.1
Firebase 또는 Supabase 검토
이메일 인증
도감 동기화
동아리 공고 원격 관리
v1.2
QR 스캔
관리자 승인형 공고 등록
익명 만족도 집계
v1.3 이후
CityBrain API 연동
운영 예시 화면
캠퍼스 경보/학생 제보
4. 개인정보 최소수집 원칙

v1.0에서는 개인정보를 거의 수집하지 않는다.

허용 가능한 정보:

닉네임 mock
학과 mock
학년 mock
로컬 도감 상태
로컬 미션 상태

v1.0에서 피해야 하는 정보:

실명
정확한 학번
전화번호
실시간 위치
개인별 식당 방문 기록 서버 저장
민원 신고자 신원
교수님 개인 연구실 상세 공개
학생별 공개 랭킹
5. 위치정보 처리 원칙

배재Pick 2.0은 v1.0에서 실시간 위치 추적을 하지 않는다.

학과 나섬이 투어와 미션은 QR 또는 코드 기반으로 설계한다.

사용자의 이동 동선을 서버에 저장하지 않는다.

6. 학생식당 개인정보 원칙

학생식당 기능은 개인정보 리스크를 낮게 유지한다.

v1.0에서 제공하는 기능:

오늘 메뉴 조회
혼잡도 mock 확인
메뉴 카드 획득
만족도 버튼 UI

서버 저장 제외:

개인별 식당 방문 기록
개인별 메뉴 선호 기록
위치 기반 방문 기록
7. 학교명 / 나섬이 사용 주의

배재대학교 명칭과 나섬이 캐릭터 사용은 공식 출시 전 학교 측 확인이 필요하다.

v1.0은 다음 중 하나로 운영한다.

내부 테스트용 MVP
학교 제안용 데모
비공식 베타 버전
학교 승인 후 정식 배포 버전
8. 결론

배재Pick 2.0은 초기에는 서버비와 개인정보 리스크를 줄이기 위해 로컬 저장 중심으로 개발한다.

실제 인증, 서버 동기화, 관리자 기능, 신고 기능은 사용자 반응과 학교 협의 이후 단계적으로 도입한다.

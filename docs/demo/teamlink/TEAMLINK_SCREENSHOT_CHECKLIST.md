# TeamLink Screenshot Checklist

## Required Screenshots

- [ ] TeamLink main screen
- [ ] Official notice source card
- [ ] Role tag area
- [ ] Contest notice cards
- [ ] Team recruitment cards
- [ ] Recruitment detail screen
- [ ] Create recruitment mock screen
- [ ] Trust / safety policy screen
- [ ] Report reason chips

## Recommended Capture Order

1. TeamLink main
2. Contest notice area
3. Team recruitment area
4. Recruitment detail
5. Create recruitment mock
6. Trust / safety policy

## Capture Rules

- Use real phone if possible
- Use portrait mode
- Use light mode
- Keep status bar visible
- Hide personal notifications
- Do not crop important UI
- Capture full screen
- Use same device for all screenshots

## Screenshot Purpose

These screenshots are for:

- professor explanation
- student council explanation
- contest proposal material
- internal testing
- app feature documentation

## Review Questions

For each screenshot, check:

1. Can the viewer understand the feature within 3 seconds?
2. Is the difference between official notice and student recruitment clear?
3. Are role tags easy to understand?
4. Is there too much text?
5. Does the screen look like a real app, not just a document?
6. Is the trust/safety limitation visible enough?

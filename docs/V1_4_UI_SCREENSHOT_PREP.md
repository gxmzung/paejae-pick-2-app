# 배재Pick 2.0 v1.4 UI / Screenshot Prep

## Goal

v1.4 prepares Paejae Pick 2.0 for internal test screenshots and app presentation.

This version focuses on test flow, screenshot planning, and release-facing documentation.

## Why

Before distributing the app to testers or preparing Play Store materials, the project needs a clear screenshot and demo sequence.

The app already has the following MVP features:

- Splash / Login mock
- Home
- Cafeteria detail
- QR mission mock
- Code mission flow
- Collection / card detail
- Department Nasumi Tour
- Club notice detail
- My Page
- Privacy guide
- Local data reset

## Internal Test Demo Flow

### Flow 1: First impression

1. Open app
2. Show Splash / Login
3. Enter app
4. Show Home

### Flow 2: Cafeteria reason to install

1. Tap cafeteria card
2. Show today's menu
3. Show congestion mock
4. Acquire menu card

### Flow 3: Mission and collection loop

1. Open mission code screen
2. Tap quick code
3. Complete mission
4. Check collection screen

### Flow 4: Campus exploration

1. Open Department Nasumi Tour
2. View department cards
3. Move to code input
4. Collect department Nasumi

### Flow 5: Club notice hall

1. Open club tab
2. Tap club card
3. View recruitment details
4. Save favorite club
5. Check My Page

### Flow 6: Privacy-light design

1. Open My Page
2. Open privacy guide
3. Show local storage policy
4. Show local data reset

## Screenshot Rule

Screenshots should show the app as a useful campus app, not only a cute collection app.

The strongest order is:

1. Student cafeteria
2. Mission / collection
3. Department tour
4. Club notice
5. Privacy
